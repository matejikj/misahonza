# 3) Motion specifikace pro GSAP/ScrollTrigger (k předání vývoji)

Níže je **produkční specifikace**, ne čistý pseudokód. Vývojář podle toho ví:

* co pinovat
* kde scrub
* breakpoints
* fallback

---

## 3.1 Obecná pravidla motion systému

### Knihovny

* `GSAP`
* `ScrollTrigger`
* `Lenis` (pouze homepage)
* volitelně `SplitType` (pokud chcete line reveal textu)

### Výkonová pravidla

* **Pinned sekce max 2** na desktopu (Materiály + Realizace)
* Na mobilu (`<768`) vypnout pin pro všechny těžké sekce
* Preferovat `transform` + `opacity`
* Vyhnout se animaci `filter: blur()` ve velkém rozsahu
* Obrázky optimalizovat (Astro image pipeline + sizes/srcset)

### Breakpointy (doporučeno)

* **mobile:** `0–767`
* **tablet:** `768–1023`
* **desktop:** `1024+`
* **large desktop:** `1440+` (případně upravit durations)

### Scroll behavior

* Lenis zapnout na homepage only
* Pro `ScrollTrigger` použít scrollerProxy (pokud integrujete Lenis)
* `scrub` používat u story sekcí, ne u všeho

### Reduced motion

Pokud `prefers-reduced-motion: reduce`:

* vypnout pinning a scrub animace
* ponechat jen jednoduché fade-in/opacity (nebo nic)

---

## 3.2 Hero Motion spec

### Cíl

Jemný cinematic nástup bez přehnaného efektu.

### Elementy

* `.hero-bg`
* `.hero-image`
* `.hero-title-line`
* `.hero-sub`
* `.hero-cta`
* `.hero-trust`

### Chování

#### On load (ne ScrollTrigger, klasický timeline)

* title lines: `y: 24 -> 0`, `opacity: 0 -> 1`, stagger 0.08
* sub: `y: 16 -> 0`, `opacity: 0 -> 1`
* CTA row: `y: 12 -> 0`, `opacity: 0 -> 1`
* trust row: `opacity: 0 -> 1`
* hero-image: `scale: 1.03 -> 1`, `opacity: 0 -> 1`

**Duration:** 0.8–1.2s
**Ease:** `power2.out`

#### Scroll-linked subtle parallax (desktop/tablet)

**Trigger:** `.hero`
**Start:** `top top`
**End:** `bottom top`
**Scrub:** `0.4`

* `.hero-image`: `yPercent: 0 -> 8`
* `.hero-bg`: `yPercent: 0 -> 4`
* `.hero-text-wrap`: `y: 0 -> -20`, opacity lehce 1 -> 0.92

### Mobile

* Bez parallax textu
* Maximálně image `yPercent 0 -> 4`

## 3.3 Materiály — Stack stěrek (pinned) ✅

### Cíl

„Vzorkovník“ efekt: karty najíždějí z prava, vrství se a mění aktivní obsah vlevo.

### DOM struktura (doporučení)

* `.materials-section`

  * `.materials-sticky` (pinned wrapper)

    * `.materials-copy-panel` (left)
    * `.materials-stack-stage` (right)

      * `.sample-card[data-index="0"...]` ... `[5]`

### Desktop / tablet (1024+ doporučeno, tablet volitelně zjednodušit)

#### ScrollTrigger (master)

**Trigger:** `.materials-section`
**Start:** `top top`
**End:** `+=220%` *(podle počtu karet; pro 6 karet klidně 240%)*
**Pin:** `.materials-sticky`
**Pin spacing:** true
**Scrub:** `0.6`

### Timeline logika (6 karet)

Každá karta má svůj „step“ (např. 0.16 z celku timeline).

#### Výchozí stav karet

* `xPercent: 30`
* `y`: index * 12 px
* `rotationZ`: `0.5–1deg`
* `scale`: `0.96`
* `opacity`: `0`
* `zIndex` podle pořadí

#### Pro každou kartu (step i)

1. **Vjezd karty**

   * `xPercent: 30 -> 0`
   * `opacity: 0 -> 1`
   * `scale: 0.96 -> 1`
   * `rotationZ: 1deg -> 0deg`

2. **Stack positioning předchozích**

   * starší karty:

     * `x: - (i * 8px)`
     * `y: + (i * 10px)`
     * `scale: 1 -> 0.97 / 0.95`
     * `opacity: 1 -> 0.75 / 0.55`

## 3.6 Realizace — Flip Gallery 3x (pinned) ✅

### Cíl

Během scrollu proběhnou 3 realizace přes 3D flip.

### Důležitá poznámka UX

3D flip je super na desktopu, ale:

* dělat ho **jen 1 karta + 3 obsahové stavy**
* držet čisté tempo (ne moc rychlé)
* na mobilu vypnout

### DOM struktura (doporučení)

* `.projects-flip-section`

  * `.projects-intro`
  * `.projects-pin-wrap`

    * `.flip-stage`

      * `.flip-card`

        * `.flip-face.front`
        * `.flip-face.back`
    * `.project-meta`
    * `.project-progress`

### Desktop motion spec

#### ScrollTrigger (master pin)

**Trigger:** `.projects-flip-section`
**Start:** `top top`
**End:** `+=260%` *(3 flippy + prostor na čtení)*
**Pin:** `.projects-pin-wrap`
**Scrub:** `0.7`

### Flip timeline (3 projekty)

Budeme mít dataset `projects[0..2]`.

#### Výchozí stav

* Card rotationY = 0
* Front = projekt 1
* Back = projekt 2

#### Segmentace progress

Celý timeline rozděl na 3 segmenty:

* Segment A: flip 1 → 2
* Segment B: flip 2 → 3
* Segment C: závěrečné „settle + CTA“

#### Segment A (0–0.33)

* `.flip-card` `rotationY: 0 -> 180`
* Ve ~90° (pomocí progress checkpointu) přepnout metadata na projekt 2
* Po dosažení 180°:

  * reset card interně na `rotationY: 0` (technicky přes state swap) **nebo**
  * pokračovat kumulativně (0→180→360) a mapovat obsah podle modulo
* Doporučuju **kumulativní rotaci** kvůli plynulosti:

  * Segment A: 0→180
  * Segment B: 180→360

#### Segment B (0.33–0.66)

* `.flip-card` `rotationY: 180 -> 360`
* Ve ~270° přepnout metadata na projekt 3

#### Segment C (0.66–1)

* Už neflipovat
* Karta se lehce „dosedne“:

  * `y: 0 -> -8 -> 0` (jemně)
  * CTA fade in
  * progress dokončení `03 / 03`

### Perspektiva / 3D

* Parent `.flip-stage` musí mít `perspective: 1200px`
* `.flip-card` `transform-style: preserve-3d`
* Faces `backface-visibility: hidden`

### Metadata sync

Metadata panel animovat zvlášť:

* `opacity: 1 -> 0 -> 1`
* `y: 0 -> 8 -> 0`
  Přepnutí při crossing checkpointu (90° / 270°)

### Mobile fallback (<768)

* Pin OFF
* 3 karty jako swipe carousel
* Scroll reveal only
* Ponechat progress `1/3`, `2/3`, `3/3`
